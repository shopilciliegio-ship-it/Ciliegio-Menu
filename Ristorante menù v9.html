⚠ SOLUZIONE ERRORE "BAD CREDENTIALS"
Il problema è quasi certamente il tipo di token. Devi usare un Personal Access Token Classic, non fine-grained.

Come crearlo correttamente:
1. Vai su GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Clicca Generate new token (classic)
3. Seleziona lo scope repo (tutta la riga — dà accesso completo ai repo privati)
4. Genera e copia il token (inizia con ghp_)

Il file JSON verrà salvato nella root del repository come specificato sotto.
Come funziona la sincronizzazione automatica:
Ogni modifica (piatto, menù, prenotazione) viene salvata su GitHub automaticamente dopo 1,5 secondi. All'apertura della pagina i dati vengono caricati da GitHub. L'indicatore in alto a destra mostra lo stato: ✓ Sincronizzato / … Salvataggio / ✕ Errore.
Per lavorare su un secondo dispositivo: apri la pagina, inserisci il token, clicca Salva config — i dati si caricano automaticamente.
